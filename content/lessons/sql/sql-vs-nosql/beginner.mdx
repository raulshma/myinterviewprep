# SQL vs NoSQL: What's the Difference?

Not all databases work the same way. Let's explore the two main types!

## ğŸ¤” The Big Question

**SQL Databases** (Relational):

- Data lives in tables with rows and columns
- Use SQL language
- Example: PostgreSQL, MySQL, SQL Server

**NoSQL Databases** (Non-Relational):

- Data can be stored in many different ways
- Each type has its own way of querying
- Example: MongoDB, Redis, Cassandra

## ğŸ“¦ A Simple Analogy

Think of storing information about your friends:

### SQL Approach: Filing Cabinet

```
Friends Table:
| ID | Name    | Phone       | City       |
|----|---------|-------------|------------|
| 1  | Alice   | 555-0101    | New York   |
| 2  | Bob     | 555-0102    | Los Angeles|
```

Every friend has the SAME columns. Very organized!

### NoSQL Approach: Box of Index Cards

```
Card 1: { name: "Alice", phone: "555-0101", city: "New York" }
Card 2: { name: "Bob", phone: "555-0102", pets: ["Dog", "Cat"] }
Card 3: { name: "Carol", email: "carol@email.com" }
```

Each card can have DIFFERENT information. More flexible!
